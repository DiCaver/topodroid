[3] FENETRE PéRIPHéRIQUES

Avant de vous lancer dans un relevé topographique, vous devez connecter et apairer votre appareil Android avec votre périphérique et l'étalonner. Appuyez sur le bouton Périphériques dans la fenètre principale afin d'ouvrir la Fenêtre périphériques La liste des périphériques connus est affichée. Les périphériques supportés sont les suivants : DistoX, DistoX2, DistoX-BLE, SAP5 et Bric4

TopoDroid n'utilise qu'un seul appareil à la fois. Pour le sélectionner, appuyez sur son nom dans la liste. Le nom du périphérique sélectionné et affiché en hat de l'écran, sous les boutons. Si aucun appareil n'est sélectionné, un message rouge s'affiche : "Aucun périphérique sélectionné".

Si la liste des périphériques est vide, votre périphérique à besoin d'être appairé avec votre appareil Android. Cela peut être réalisé avec l'application Paramètres d'Android.
Si votre périphérique ne s'apaire pas automatiquement, le code PIN des DistoX est 0000 (quatre zéros). Le code Pin des SAP5 est "000000" (six zéros).

Nom du périphérique
Par défaut, TopoDroid utilisé l'adress MAC bluetooth comme nom de périphérique. Les périphériques peuvent être identifiés par un alias pour les distinguer entre eux plus facilement.

Nota : Les boutons et menus marqués d'une [*] sont spécifique aux DistoX et nécessite que le périphérique soit allumé.

ACTIONS

  • appuyez sur la ligne d'un périphérique le sélectionne pour en faire l'appareil à utiliser.
  • Un appuis long sur un périphérique permet de définir le alias ou définir un "second" DistoX

BOUTONS

  • Réinitialise la connexion bluetooth.
  • Information du périphérique[N,*]
  • Fonctions mémoire des DistoX ou Bric4[*]
  • Active/désactive le mode d'étalonnage du DistoX [*]
  • Etalonnages des DistoX
  • Lit et affiche et les coefficients d'étalonnage depuis le DistoX [A,*]

Pour le DistoX tous les boutons sont actifs.
Pour le Bric4, les trois premiers boutons sont actifs.
Pour le SAP5, seul le premier bouton est actif
Si la bascule du DistoX en mode étalonnage échoue au premier essai, répétez l'opération une seconde fois.

La lecture du périphérique échoue si TopoDroid est connecté au périphérique et télécharge des données. Cela peut arriver quand la Fenêtre périphériques est ouverte depuis la Fenêtre des visées.

MENUS

  • Déconnecte déconnecte le périphérique actif [A]
  • Télécharge mes firmwares vers le DistoX (v. 2 and BLE) [E,*]
  • Affiche les journaux des paquets du DistoX[T]
  • Paramètres
  • Aide

DEVICE SETTINGS

  • Bluetooth: whether to enable, disable, or check on start-up [default "check"]
  • Number of new data: get the number of new data before downloading them (on-demand mode) [default "no"]

The following settings apply only to the DistoX (which uses classical bluetooth).

  • Connection mode. "On-demand", "continuous", or "multi" [default "on-demand"]
  • DistoX Socket type. Normal or "insecure" [default is "insecure"]
  • DistoX Z6-workaround: a download timeout error on Z6 phone, and possibly other devices. It is safe to leave this enabled [default "on"]

Connection mode
There are three connections modes for the DistoX.

  • in on-demand mode data are downloaded in burst, and when there is no more data to download the connection is closed.
  • in continuous mode TopoDroid remains connected to the DistoX and the shots are downloaded as soon as they are taken. If the DistoX and Android disconnect, TopoDroid repeatedly tries to reconnect. Otherwise, the bluetooth connection is closed.
  • the multi mode is like the "on-demand" mode, but a long-tap on the "download" button, in the Shot window or in the Sketch window, changes the DistoX. It you have set a "second" DistoX (in the Device window) it is swapped with the current DistoX. Otherwise, you get a dialog to pick one among the paired DistoXes.

With the "multi" mode, the current DistoX is shown in the title of the Shot and Sketch windows.
The connection mode applies only to the data download. Other DistoX functions use a one-shot query mode.

Sap5 and Bric4 are always connected in continuous mode.

Custom settings: Connection delay, Second DistoX, Data pause, Data-ready wait, Laser pause, Shot pause.
