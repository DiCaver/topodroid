[3] DEVICE WINDOW

Before surveying you need to connect and pair Android with the DistoX and calibrate it. Tap the DistoX button in the main window to open the Device window. The list of known DistoX devices is shown in the middle. TopoDroid works with one DistoX at a time. To select it tap on its entry in the list. The name of the selected DistoX is shown at the top, just below the buttons. If no DistoX is selected, there is a red message saying "Device not selected".

If the DistoX list is empty, your DistoX needs to be connected and paired with the Android. This can be done with TopoDroid (Scan and Pair menus) or with the Settings app.

DistoX nickname
By default TopoDroid uses the bluetooth address as name for a DistoX. The DistoXes can be given nicknames to distinguish among several devices more easily.

Note. Buttons and menus marked with [*] need the DistoX on.

ACTIONS

  • tap on a DistoX makes it the active DistoX
  • long tap on a DistoX sets the nickname or set the "second" DistoX

BUTTONS

  • reset bluetooth connection.
  • toggle the DistoX calibration mode on and off [*]
  • Calibrations of the active device.
  • DistoX information [A,*]
  • read and display the calibration coefficients from the DistoX [A,*]
  • DistoX memory functions [*]

MENUS

  • Scan for new DistoX devices [N,*]. You might have to try two or three times before the DistoX is found
  • Pair the active DistoX with the Android, if not already paired [N,*]
  • Disconnect clears the active device (the DistoX remains paired) [A]
  • Dump and upload firmwares to the DistoX2 [E,*]
  • Display log of DistoX packets [T]
  • Settings
  • Help

DEVICE SETTINGS

  • Bluetooth: whether to enable, disable, or check on start-up [default "check"]
  • Connection mode: either "on-demand", "continuous", or "multi" [default "on-demand"]
  • Auto reconnect: automatically reconnect if the connection is lost in "continuous" data download-mode [default "no"]
  • Number of new data: get the number of new data before downloading them (on-demand mode) [default "no"]
  • Socket type: either normal or "insecure" [default is device-dependent]
  • Z6-workaround Fixes a download timeout error on a Z6 phone, and possibly other devices. It is safe to leave this enabled [default "on"]
  • Auto pairing: whether TopoDroid should try to pair with the DistoX selected from a scan [default "no"]

Connection modes
There are three connections modes.

  • in on-demand mode data are downloaded in burst, and when there is no more data to download the connection is closed.
  • in continuous mode TopoDroid remains connected to the DistoX and the shots are downloaded as soon as they are taken. When the DistoX and Android get disconnected, TopoDroid repeatedly tries to reconnect if the auto-reconnect option is enabled, Otherwise, the bluetooth connection is closed.
  • the multi mode is like the "on-demand" mode, but a long-tap on the "download" button, in the Shot window or in the Sketch window, changes the DistoX. It you have set a "second" DistoX (in the Device window) it is swapped with the current DistoX. Otherwise, you get a dialog to pick one among the paired DistoXes.

With the "multi" mode, the current DistoX is shown in the title of the Shot and Sketch windows.
The connection mode applies only to the data download. Other DistoX functions use a one-shot query mode.

Custom settings: Connection delay, Second DistoX, Data pause, Data-ready wait, Laser pause, Shot pause.
